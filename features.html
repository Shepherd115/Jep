Ominaisuudet


Tämä ei sisällä ominaisuuksia jotka ovat tulleet osaksi näitä infrastruktuuri projekteja mihin GrapheneOS nojautuu (AOSP, LLVM, OpenBSD) tämä myöski on VAIN ominaisuudet jota GrapheneOS lisää jo valmiiseen AOSPn turvallisuusmalliin. AOSP on jo valmiiksi erittäin turvallinen kuten sisältää koko systeemiä koskevat SElinux säännöt, verified-bootin, muistiturvallisten ohjelmointikielien käytön, (Java / Kotlin) CFI, SCS ja kunnollisen sovellusten hiekkalaatikon

GrapheneOS on turvallinen ja yksityinen mobiilikäyttöjärjestelmä hyvällä käytettävyydellä. Se aloittaa vahvasta Android Open Source Projectin (AOSP) turvallisuumallista ja varoo tämän rikkomista tai hyökkäyspinnan lisäämistä. GrapheneOS säilyttää kaikki Androidin tarjoamat turvallisuus ominaisuudet ja kyvyn lukita bootloader asennuksen jälkeen antaen täyden verified-bootin. GrapheneOS keskittyy oleelliseen markkinoinnin sijan, se ei ota tyypillistä käytäntöä asettaa satunnaisia parannuksia ja ominaisuuksia vain koska ne näyttävät hyvältä. Se on erittäin tekninen projekti rakentaen turvallisuutta ja yksityisyyttä itse käyttöjärjestelmään.

Muut omiaisuudet jotka GrapheneOS lisää, katso linkit yksityiskohtiin, tämä on vain yhteenveto!:
  • Vahvistettu / Hardened memory allocator
    • Out-Of-Line meatiedot sekä suojaus korruptoitumiselta estäen yleiset hyväksikäytöt
    • Erilliset muistialueet metatiedoille, suurille varauksille sekä pienille eikä osoitetilaa koskaan uudelleenkäytetä muilla alueilla kuin heidän omallaan
    • Virheellisten vapautusten determistinen havaitseminen
    • Alustus vapautuksen yhteydessä sekä kirjoitus-vapautuksen-jälkeen havaitseminen tarkistamalla että muisti on silti vapautettu ennen kuin se annetaan käyttöön uudelleen
    • Viivästetty osoitetilan uudelleenkäyttö
    • Korkean entropian satunnaisen kokoiset vartija alueet suurien varausten ympärillä
    • Canary-arvoja lisätään pienempiin varauksiin
  • Vahvistettu / Hardened bionic-c standardikirjasto joka kulkee käsi-kädessä hardened_mallocin kanssa
  • Vahvistettu / Hardened sovellusten hiekkalaatikko / sandbox
  • Tiukennetut SElinux säännöt
  • Vahvistettu / Hardened app-runtime
  • Vahvistettu / Hardened ohjelmointikielen kääntäjä (compiler toolchain)
  • Vahvistettu / Hardened kernel
    • Dynaamisesti ladattavat kernel moduulit on poistettu käytöstä parantamaan CFIn rakeisuutta ja pienentämään hyökkäyspintaa
    • 4-level-page-tables käytössä arm64 alustalla antamaan huomattavasti suuremman entropian osoitetila ja ASLR
    • Canary-arvoja asetetaan kernelin kasaan (heap)
    • Muisti pyyhitään (nollataan) heti vapautuksen jälkeen
    • Mahdollisuus kytkeä perf ja ptrace pois päältä hyökkäyspinnan vähentämiseksi
  • Eng:Prevention of dynamic native code execution in-memory or via the filesystem for the base OS without going via the package manager, etc.
  • Eng:Filesystem access hardening
  • Paranneltu laitteisto pohjainen vahvistus tarkemmille laitteisto tiedoilla
  • Eliminoi viimeisetkin kolot mistä sovellukset voisivat saada pääsyn laitetunnistimiin
  • Huomattavasti pienennetty hyökkäyspinta poistamalla turhaa koodia, tekemällä useammasta ominaisuudesta vaihtoehtoisen sekä poistamalla vaihtoehtoisia ominaisuuksia oletuksena (NFC, Bluetooth, Wifi jne)
  • Vaihtoehto kytkeä ptrace pois päältä hyökkäyspinnan pienentämiseksi
  • Pieniä parannuksia laitteen salaukseen
  • Tuki kirjautua ulos profiileista ilman laitteen uudelleenkäynnistämistä. Tämä tuhoaa profiilin salausavaimet ja asettaa profiilin takaisin lepoon
  • Vaihtoehto asettaa laite käynnistymään uudelleen haluaman ajan jälkeen asettaen laitteen BFU (Before First Unlock) tilaan
  • Vaihtoehto sekoittaa PIN näppäimistön numeroiden järjestys
  • LTE-only vaihtoehto poistaen jättimäisiä määriä vanhentunutta koodia pienentäen hyökkäyspintaa
  • Satunnaistettu MAC osoite joka kerta kun laite etsii wifi verkkoja (Käytössä oletuksena)
  • Kun satunnaistettu MAC osoite on käytössä DHCP tila pyyhitään joka kerta ettei verkko voi tunnistaa että laite on sama kuin viimeksi
  • Parannellut ipv6 yksityisyys osoitteet estäen seurantaa verkkojen välillä
  • Vanadium joka on vahvistettu forkki chromium selaimesta sekä laitteen webviewin tarjoaja
  • Auditor sovellus vahvistamaan laitteen identiteetti sekä käyttöjärjestelmän autenttisuus ja koskemattomuus
  • PDF lukija, simppeli, sandboxattu, ei tarvitse lupia
  • Salatut varmuuskopiot seedvault-backup sovelluksen avulla
  • Turvallinen sovellusten laukasu malli (Exec)
  • Kyky evätä haluamiltaan sovelluksilta internet lupa estäen suorat ja ei suorat yhteydet sovellukselta. Localhost on myös suojattu tämän avulla. Toisinkuin palomuuri tyylinen asettelu, Network kytkin estää sovellusta käyttämästä internettiä käyttöjärjestelmän tarjoamien ohjelmointirajapintojen (API) avulla joten sovellukset ei voi esimerkiksi käyttää IPCtä rajoitusten ohittamiseen
  • Kyky evätä haluamiltaan sovelluksilta Sensors pääsy mikä estää sovellusta käyttämästä laitesensoreita (kiihtyvyysanturi, gyroskooppi, kompassi, barometri, lämpömittari jne)
  • Kellon päivittäminen omien GrapheneOS palvelimien kautta
  • Kunnollinen tuki poistaakseen puhelinoperaattorin kyvyn vaihtaa laitteen kellonaikaa eikä vain olla näyttämättä heidän tarjoamaa aikaa
  • Vahvistettu / Hardened signeeraus infrastruktuuri
  • Tuki 64 merkkiselle laitteen salasanalle
  • Vaivattomat käyttöjärjestelmän päivitykset mitkä pysyvät poissa tieltä sekä tuki mennä aiempaan versioon jos päivitys epäonnistuu
  • Vaatii laitteen avausta ennen kuin sensitiiviä ominaisuuksia quick-tilesseissä voi käyttää (Wifi, lentokonetila, GPS, NFC jne)
  • Minimaalinen määrä ennalta asennettuja sovelluksia / palveluita
  • Ei Googlen palveluita (GooglePlayPalvelut) tai sovelluksia integroitu itse käyttöjärjestelmään
  • Eng: Fixes for multiple serious vulnerabilities not yet fixed upstream due to a flexible release cycle / process prioritizing security
