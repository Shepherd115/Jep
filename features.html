Ominaisuudet

GrapheneOS on turvallinen ja yksityinen mobiilikäyttöjärjestelmä hyvällä käytettävyydellä sekä toimivuudella. Se aloittaa AOSPn vahvasta turvallisuusmallista ja huolehtii ettei se satuta tätä tai lisää hyökkäyspintaa. GrapheneOS sisältää kattavan lajitelman tarkasti suunniteltuja komponentteja jotka toimivat harmoniassa parantaen muistisuojausta, sulkemaan kokonaisia exploitti luokkia sekä tekee useiden hyökkäysten suorittamisesta huomattavasti vaikeampaa. Projekti välittää erittäin paljon toimivuudesta, sekä sovellusten yhteensopivuudesta joten ne otetaan huomioon kaikissa ominaisuuksissa.

GrapheneOS on keskittynyt oleelliseen markkinoinnin sijan. Se ei ota tyypillistä käytäntöä asettaa satunnaisia epäturvallisia ominaisuuksia toivoen että hyökkääjä ei tiedä niistä. Se on erittäin tekninen projekti rakentaen yksityisyyttä ja turvallisuutta itse käyttöjärjestelmään hyödyttömien ominaisuuksien tai satunnaisten sovelluksien sijan.

Tämä lista sisältää yhteenvedon ominaisuuksista joilla GrapheneOS eroaa AOSPstä. Se ei dokumentoi useita historiallisia ominaisuuksia jotka ei ole enää käytössä syystä tai toisesta. Useimmat ominaisuudet on saatu infrastruktuuri projekteihin joihin GrapheneOS nojautuu(AOSP, Linux, LLVM).

Suuret komponentit

Vain yhteenveto joten katso linkit yksityiskohtiin

  • Vahvistettu / Hardened memory allocator joka kohottaa modernit laitteistonkyvyt tarjoamaan huomattavaa suojaa yleisiä tietoturvareikiä vastaan (Kasan muistikorruptiot) ja on siirrettävissä muihinkin Linux pohjaisiin käyttöjärjestelmään ja onkin adoptoitu muiden turvallisuus painotteisten käyttöjärjestelmien toimesta kuten Whonixin.
    • Metatiedot ovat kokonaan eristettyjä sekä suojaus korruptoitumiselta
      • allocator_staten osoiteavaruus on kokonaan varattu alustamisen aikana eikä sitä koskaan käytetä varauksille tai millekkään muulle
      • Globaaleissa muuttujissa oleva tila on kokonaan read-only alustamisen jälkeen sekä osoittajat osoittavat eristettyyn allocator_stateen joten kirjaston osoitteen vuotaminen ei paljastaisi kirjoitettavan tilan osoitetta
      • allocator_state sijaitsee omassa alueessaan korkean entropian vartija-alueiden ympäröimänä
      • Suojaus MPKn kautta (Memory Protection Keys) x86_64 suoritinarkkitehtuurilla (Oletuksena ei käytössä)
      • [TULEVA] Suojaus MTEn kautta ARMv8.5+ pohjalla
    • Virhellisten varausten determistinen havaitseminen
    • Zero-on-free sekä write-after-free havaitseminen tarkistamalla että muisti on silti nollattu ennen sen uudelleenkäyttämistä
    • Viivästetty osoittilan uudelleenkäyttö FIFOn ja randomisoinnin avulla
    • Hienojakoinen randomisointi
    • Aggressiiviset järjestystarkastukset
    • Muistisuojatut vartija-alueet yli 16k kokoisten varausten ympärillä sekä vartija-alueiden kokojen randomisointi (128k ja korkeampi)
    • Alle 16k kokoisiset varaukset saavat vartija-alueen jokaisen slabsin väliin joka sisältää varauksia
    • Satunnaisia canary-arvoja lisätään pieniin varauksiin havaitsemaan ja estämään muistinylivuoto tilanteita ja muita kasan vioittumisia kun canary arvo tarkistetaan (Pääosin vapautuksen yhteydessä)
  • Vahvistettu / Hardened bionic-c standardikirjasto joka kulkee käsi-kädessä hardened_mallocin kanssa ja tarjoaa suojaa muistinkorruptio ongelmia vastaan.
  • Vahvempi sovellusten hiekkalaatikko / sandbox
  • Tiukennetut SElinux säännöt
  • Vahvistettu / Hardened app runtime (ART)
  • Vahvistettu / Hardened ohjelmointikielen kääntäjä (Compiler)
  • Vahvistettu / Hardened kernel
    • Tuki dynaamisesti ladattaville kernelmoduleille on kytketty pois ja ainoastaan tietty sarja moduleja laitteelle on rakennettu kerneliin parantamaan CFItä ja pienentämään hyökkäyspintaa
    • 4 tasoiset sivutaulukot on otettu käyttöön antamaan huomattavasti suuremman entropian osoiteavaruuden (48 bittiä 39 sijan) ja huomattavasti suuremman entropian ASLR (33 bittiä 24 sijan)
    • Satunnaisia canaryarvoja asetetaan kernelin kekoon estämään yli ja alivuototilanteita
    • Muisti pyyhitään heti kun se on vapautettu molemmissa alemman tason kernel page varaajassa ja korkeamman tason keon varaajassa
    • Kernel pinon varaukset nollataan
    • Hyökkäyspinnan pienennys mahdollistamalla tiettyjen ominaisuuksien kytkemisen pois (perf, ptrace)
    • Useita upstream vahvistuksia käytössä
  • Vanadium joka on vahvistettu forkki chromium selaimesta sekä laitteen webviewin tarjoaja
  • Auditor sovellus vahvistamaan laitteen identiteetti sekä käyttöjärjestelmän autenttisuus ja koskemattomuus

Muut Ominaisuudet

  • Tarkemmat laitteistotiedot asetuksissa
  • Eliminoi viimeisetkin kolot mistä sovellukset voisivat saada pääsyn laitetunnistimiin
  • Huomattavasti pienennetty hyökkäyspinta poistamalla turhaa koodia, tekemällä useammasta ominaisuudesta vaihtoehtoisen sekä poistamalla vaihtoehtoisia ominaisuuksia oletuksena (NFC, Bluetooth, Wifi jne)
  • Vaihtoehto kytkeä ptrace pois päältä hyökkäyspinnan pienentämiseksi
  • Pieniä parannuksia laitteen salaukseen
  • Tuki kirjautua ulos profiileista ilman laitteen uudelleenkäynnistämistä. Tämä pyyhkii profiilin salausavaimet muistista ja asettaa profiilin takaisin lepoon
  • Vaihtoehto asettaa laite käynnistymään uudelleen haluaman ajan jälkeen asettaen laitteen BFU (Before First Unlock) tilaan
  • Vaihtoehto sekoittaa PIN näppäimistön numeroiden järjestys
  • LTE-only vaihtoehto poistaen jättimäisiä määriä vanhentunutta koodia pienentäen hyökkäyspintaa
  • Satunnaistettu MAC osoite joka kerta kun laite etsii wifi verkkoja (Käytössä oletuksena)
  • Kun satunnaistettu MAC osoite on käytössä DHCP tila pyyhitään joka kerta ettei verkko voi tunnistaa että laite on sama kuin viimeksi
  • Parannellut ipv6 yksityisyys osoitteet estäen seurantaa verkkojen välillä
  • PDF lukija, simppeli, sandboxattu, ei tarvitse lupia
  • Salatut varmuuskopiot seedvault-backup sovelluksen avulla
  • Moderni mikrofoni/kamera indikaattori UX myös sijainnille
  • Tiukempi implementaatio vaihtoehtoisesta sormenjälki avaamisesta jossa ainoastaan 5 yritystä sallitaan 20 sijan ennen pysyvää lukitsemista
  • Paranneltu verified-boot paremmilla turvallisuusominaisuuksilla ja pienennetyllä hyökkäyspinnalla (kokonainen dex preopt käyttöjärjestelmän sovelluksille, SElinux vahvistuksia, useiden hyödyttömien ominaisuuksien poistamista kuten system_other odex ja paljon muutakin)
  • Turvallinen sovellusten laukaisu malli (Exec)
  • Network lupakytkin estämään molemmat suorat ja ei suorat pääsyt verkkoihin. Localhost on myös suojattu tämän avulla mikä on tärkeää estämään sovelluksia kommunikoimasta käyttäjäprofiilien välillä. Toisinkuin palomuuri tyylinen asettelu, Network lupakytkin estää sovelluksia käyttämästä verkkoja käyttöjärjestelmän APIen kautta tai muiden sovellusten kunhan ne ovat merkitty oikein
  • INTERNET lupa jota käytetään Network lupakytminen perustana on vahvistettu ja kunnollinen tuki mahdollistaen estämisen/sallimisen per-profiili kohtaisesti
  • Kyky evätä haluamiltaan sovelluksilta Sensors pääsy mikä estää sovellusta käyttämästä laitesensoreita (kiihtyvyysanturi, gyroskooppi, kompassi, barometri, lämpömittari jne)
  • Kellon päivittäminen omien GrapheneOS palvelimien kautta
  • Oma uusi ja moderni kamerasovellus
  • Kunnollinen tuki poistaakseen puhelinoperaattorin kyvyn vaihtaa laitteen kellonaikaa eikä vain olla näyttämättä heidän tarjoamaa aikaa
  • Vahvistettu / Hardened signeeraus infrastruktuuri
  • Pieniä muutoksia oletuksiin jotka suosivat yksityisyyttä käytännöllisyyden yli kuten sensitiivinen tieto on piilotettu lukitusnäytöltä, salasanoja ei näytetä hetkellisesti niitä kirjoittaessa jne
  • Googleplaypalvelu yhteensopivuuskerros
  • Tuki 64 merkkiselle laitteen salasanalle
  • Vaivattomat käyttöjärjestelmän päivitykset mitkä pysyvät poissa tieltä sekä tuki mennä aiempaan versioon jos päivitys epäonnistuu
  • Vaatii laitteen avausta ennen kuin sensitiiviä ominaisuuksia quick-tilesseissä voi käyttää (Wifi, lentokonetila, GPS, NFC jne)
  • Minimaalinen määrä ennalta asennettuja sovelluksia / palveluita
  • Ei Googlen palveluita (GooglePlayPalvelut) tai sovelluksia integroitu itse käyttöjärjestelmään
  • Eng: Fixes for multiple serious vulnerabilities not yet fixed upstream due to a flexible release cycle / process prioritizing security
  • Eng:Prevention of dynamic native code execution in-memory or via the filesystem for the base OS without going via the package manager, etc.
  • Eng:Filesystem access hardening
  • Eng:Support for using the fingerprint scanner only for authentication in apps and unlocking hardware keystore keys by toggling off support for unlocking.
