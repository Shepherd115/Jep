Ominaisuuksien Yhteenveto

GrapheneOS on yksityinen ja turvallinen käyttöjärjestelmä hyvällä toimivuudella. Se aloittaa vahvasta AndroidOpenSourceProjectin (AOSP) turvallisuusmallista ja varoo tämän rikkomista tai hyökkäyspinnan lisäämistä. GrapheneOS tekee huomattavia parannuksia molempiin yksityisyyteen sekä turvallisuuteen useiden tarkasti suuniteltujen ominaisuuksien avulla jotka on tarkoitettu toimimaan realistisia uhkia vastaan. Projekti välittää erittäin paljon toimivuudesta ja sovellusten yhteensopivuudesta joten ne otetaan huomioon kaikissa ominaisuuksissa.

GrapheneOS keskittyy oleelliseen markkinoinnin sijan. Se ei ota tyypillistä käytäntöä asettaa satunnaisia ominaisuuksia toivoen että vastustaja ei tiedä niistä. Se on erittäin tekninen projekti rakentaen yksityisyyttä ja turvallisuutta itse käyttöjärjestelmään hyödyttömien ominaisuuksien sijan.

Tämä sivu sisältää yhteenvedon ominaisuuksista jolla GrapheneOS eroaa AOSPstä. Se ei dokumentoi useita historiallisia ominaisuuksia jotka ei ole käytössä enään syystä tai toisesta. Useimmat ominaisuudet on saatu projekteihin joihin GrapheneOS tukeutuu kuten AOSP, Linux ja LLVM.

On myös hyvä huomioida että GrapheneOS käyttää Androidia tällä hetkellä pohjanaan ja AOSP on jo valmiiksi erittäin turvallinen kuten sisältää koko systeemiä koskevat SElinux säännöt, verified-bootin, muistiturvallisten ohjelmointikielien käytön, (Java / Kotlin) CFI, SCS ja kunnollisen sovellusten sandboxin. Android 12 myös toi suuria parannuksia.

Muut Ominaisuudet

  • Tarkemmat laitteistotiedot asetuksissa
  • Eliminoi viimeisetkin kolot mistä sovellukset voisivat saada pääsyn laitetunnistimiin
  • Huomattavasti pienennetty hyökkäyspinta poistamalla turhaa koodia, tekemällä useammasta ominaisuudesta vaihtoehtoisen sekä poistamalla vaihtoehtoisia ominaisuuksia oletuksena (NFC, Bluetooth, Wifi jne)
  • Vaihtoehto kytkeä ptrace pois päältä hyökkäyspinnan pienentämiseksi
  • Pieniä parannuksia laitteen salaukseen
  • Tuki kirjautua ulos profiileista ilman laitteen uudelleenkäynnistämistä. Tämä tuhoaa profiilin salausavaimet ja asettaa profiilin takaisin lepoon
  • Vaihtoehto asettaa laite käynnistymään uudelleen haluaman ajan jälkeen asettaen laitteen BFU (Before First Unlock) tilaan
  • Vaihtoehto sekoittaa PIN näppäimistön numeroiden järjestys
  • LTE-only vaihtoehto poistaen jättimäisiä määriä vanhentunutta koodia pienentäen hyökkäyspintaa
  • Satunnaistettu MAC osoite joka kerta kun laite etsii wifi verkkoja (Käytössä oletuksena)
  • Kun satunnaistettu MAC osoite on käytössä DHCP tila pyyhitään joka kerta ettei verkko voi tunnistaa että laite on sama kuin viimeksi
  • Parannellut ipv6 yksityisyys osoitteet estäen seurantaa verkkojen välillä
  • PDF lukija, simppeli, sandboxattu, ei tarvitse lupia
  • Salatut varmuuskopiot seedvault-backup sovelluksen avulla
  • Moderni mikrofoni/kamera indikaattori UX myös sijainnille
  • Tiukempi implementaatio vaihtoehtoisesta sormenjälki avaamisesta jossa ainoastaan 5 yritystä sallitaan 20 sijan ennen pysyvää lukitsemista
  • Paranneltu verified-boot paremmilla turvallisuusominaisuuksilla ja pienennetyllä hyökkäyspinnalla (kokonainen dex preopt käyttöjärjestelmän sovelluksille, SElinux vahvistuksia, useiden hyödyttömien ominaisuuksien poistamista kuten system_other odex, exec spawning ja paljon muutakin)
  • Turvallinen sovellusten laukaisu malli (Exec)
  • Network lupakytkin estämään molemmat suorat ja ei suorat pääsyt verkkoihin. Localhost on myös suojattu tämän avulla mikä on tärkeää estämään sovelluksia kommunikoimasta käyttäjäprofiilien välillä. Toisinkuin palomuuri tyylinen asettelu, Network lupakytkin estää sovelluksia käyttämästä verkkoja käyttöjärjestelmän APIen kautta tai muiden sovellusten kunhan ne ovat merkitty oikein
  • INTERNET lupa jota käytetään Network lupakytminen perustana on vahvistettu ja kunnollinen tuki mahdollistaen estämisen/sallimisen per-profiili kohtaisesti
  • Kyky evätä haluamiltaan sovelluksilta Sensors pääsy mikä estää sovellusta käyttämästä laitesensoreita (kiihtyvyysanturi, gyroskooppi, kompassi, barometri, lämpömittari jne)
  • Kellon päivittäminen omien GrapheneOS palvelimien kautta
  • Kunnollinen tuki poistaakseen puhelinoperaattorin kyvyn vaihtaa laitteen kellonaikaa eikä vain olla näyttämättä heidän tarjoamaa aikaa
  • Vahvistettu / Hardened signeeraus infrastruktuuri
  • Pieniä muutoksia oletuksiin jotka suosivat yksityisyyttä käytännöllisyyden yli kuten sensitiivinen tieto on piilotettu lukitusnäytöltä, salasanoja ei näytetä hetkellisesti niitä kirjoittaessa jne
  • Googleplaypalvelu yhteensopivuuskerros
  • Tuki 64 merkkiselle laitteen salasanalle
  • Vaivattomat käyttöjärjestelmän päivitykset mitkä pysyvät poissa tieltä sekä tuki mennä aiempaan versioon jos päivitys epäonnistuu
  • Vaatii laitteen avausta ennen kuin sensitiiviä ominaisuuksia quick-tilesseissä voi käyttää (Wifi, lentokonetila, GPS, NFC jne)
  • Minimaalinen määrä ennalta asennettuja sovelluksia / palveluita
  • Ei Googlen palveluita (GooglePlayPalvelut) tai sovelluksia integroitu itse käyttöjärjestelmään
  • Eng: Fixes for multiple serious vulnerabilities not yet fixed upstream due to a flexible release cycle / process prioritizing security
  • Eng:Prevention of dynamic native code execution in-memory or via the filesystem for the base OS without going via the package manager, etc.
  • Eng:Filesystem access hardening
  • Eng:Support for using the fingerprint scanner only for authentication in apps and unlocking hardware keystore keys by toggling off support for unlocking.
